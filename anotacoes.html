Instalação

instalar o create-react-app: bíblioteca que a galera do face liberou para facilitar a instalação e utilização do react

npm install -g create-react-app

-- Para criar a aplicação rode esse comando:

create-react-app "nome da aplicação"

Comandos utilitários

npm start -- Inicia o projeto

npm run build -- Disponibiliza o projeto para produção

npm run teste -- Executa os testes

npm run eject -- Remove todo o projeto e deixa limpo para ser usado do 0, não pode reverter.

Cliclo de vida

ComponetWillMount (){} Função que é chamada antes da função Render ser chamada.

ComponentdidMount(){} Função que é chamada após a função Render ser chamada.

É uma boa prática trazer os dados usando essas função e não no constructor do componente, existem outros métodos do ciclo de vida do react mas esses são os principais. a variável state herdada da classe pai Componentes vai ser a mais utilizada no React, e atualizar o estado dela para os dados serem recarregados e etc vai ser algo comum, fazemos isso utilizando o método: setState fazendo assim o virtual dom ser recarregado com os novos dados de determinadas requisições.

Criação dos componentes

Ao criar um componente em react, para ter acesso aos atributos passados pela classe "pai" é preciso usar o props:

{this.props.nome}

todos os atributos são convertidos em um json, que permite o acesso em toda a classe.

Anotações

A página index.html é necessária pois, basicamente todo o código fonte que é gerado no build do projeto que se torna o bundle.js e é importado dentro do index.html, isso é feito pelo webpack.

Babel - responsável por transcrever o código javascript pra ser interpretado pelo node Js.

Webpack - fica responsável por converter nosso código que é do servidor e conseguir redenrizar ele no navegador.

High Order Components e muito praticada em projetos que utilizam o React. A ideia é que o estado fique sempre nos seus High Order Components e que os outros componentes fiquem apenas com a responsabilidade de montar a visualização em si.
